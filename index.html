Git short notes

Git Introduction -

  1. Git is Version Control System.
  2. Benefits of Git:
    1. Easy file recovery.
    2. We can easily come to know who introduced issue in our code and when.
    3. Can roll back to previosuly working states.
  3. History of git:
    1. Local version control system - DB is used to keep track of files,you can rollback to previous version,but if your hard disk is lost everything is lost.
    2. Centralized version control system - In this centralized Sever is used to keep track of files, but if you lost server ,everything is lost anyhow we can get current project back from contributors of that project but we can't rollback.
    3. Distributed version control system - This is exactly same as Centralized version control system only difference is that in this VCS every contributor has full file history so even if server got damaged than also we can rollback as everyone has full file history.
      âœ¨Git is actually Distributed Version Control System
  4. Git and Github are two different things:
    1. Git - Version Control System.
    2. Github - Hosting Service, website that host git repositories.
    3. Repository - Folder of git project.
        ex your project has files : F1,F2,F3,...Fn and has folders :Fo1,Fo2,Fo3,...,Fon so all these files and folders(.git folder) collectively called as repository.

Initial Setup -

  1. Launch git bash(it is nothing but terminal like linux, you can also use git using normal powershell windows terminal).
  2. To check git is installed or not run command git
      if it shows error means your git is not installed and if it shows something like usage , --info path means your git is installed.
  3. Git can be used in 2 ways -
    1)Command line tools (by writing commands)
    2)GUI (just by using buttons)
    But disadvantage of using GUI is that we can't use some features that can be used by using command line tools.
  4. Before using git configure your username and email as :
    1. git config --global user.name "YOUR NAME" - for configuring username
    2. git config --global user.email "YOUR EMAIL" - for configuring email
    3. git config user.name - is used to check your username
    4. git config user.email - is used to check your email

Git : Three Stages Architecture -

  1. Working Directory - Full folder on which we are working on.
  2. Staging Area - Those files which you want to be in next commit(snapshot).
  3. Git Repository - It is hidden folder with name .git which has full file history of your project.

Commands for Tracking Git Project -

  1. git status - To show status of your project,you can check if there is any changes left to be commited.
  2. git init - To make any project as git project we need to run this command.
  3. git add --a - To add all the changes into staging area at once.
  4. git commit -m "msg" - To commit changes.
  5. git log - To display information of all commits of your project.
  6. rm -rf .git - To stop tracking,It will delete .git file.

Cloning remote git reopository -

  • Search for git website <repository name >.
  • Press code button and then copy url.
  • Go in any folder and open git bash.
  • run command git clone <copied url> it will pull code of that git repository to your computer and now you will be the owner of that repository(pulled repository) and you can make any changes in that
  • Note :-You will become the owner of pulled repository not actual repository.
  • If you want to give some different name to pulled repository you can do it by running git clone <copied url> <name you want to give>

Ignoring files in git -

  1. Create .gitignore file.
  2. Add all the files and directories name which you want git should ignore them in .gitignore file.
  3. *.log - isse vo sari files ignore ho jaegi jinka extension .log hai
  4. Add <dirname>\ in .gitignore file to ignore all directories with name <dirname>.
  5. Add /dir/ to .gitignore to ignore only outter directory with name dir.
  6. Add static/dir in .gitignore file to ignore dir directory inside static folder.

git diff & git diff --stage -

git diff : Shows difference between working directory and stagging area.

git diff --stage : Show difference between previous working directory and stagging area.

Skipping stagging area -

git commit -a -m "<msg>" - is used to skip stagging area but using this only code changes will be commited not the untracked file(newly created file) for untracked file we need to separately run command git add <filename>.

Removing and renaming files using git

git rm first.txt - - Delete file with name first.txt

git mv first.txt second.txt.- Rename first.txt to second.txt.

git rm --cached - let you are atracking file than you put it into .gitignore file than also git will not ignore it bcz you were tracking it to ignore it you have to explictly mention that you want to ignore this file by running command git rm --cached.
